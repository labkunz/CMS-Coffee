# Implement-Notes Skill

在使用者確認理解後，將技術討論整理成以使用者認知為主軸的回顧筆記。

## 核心原則
- 筆記是理解的證明，不是討論的逐字稿
- 以使用者自己的描述和理解方式為主軸，搭配實際程式碼和應用情境
- 針對單一技術或邏輯，保持精簡，方便快速回顧

## 執行流程

### 1. 內容分析

- 優先抓取使用者怎麼描述這個概念的（類比、自己的話），而非 AI 的解釋
- 從討論中提取相關的程式碼範例和應用情境
- 識別討論中浮出的注意事項或易混淆的點

### 2. 檔案命名

- 使用英文 kebab-case 格式：`topic-name.md`
- 範例：`react-hooks-guide.md`、`git-workflow.md`
- 若主題是中文，轉換為英文或拼音
- 避免空格和特殊符號

### 3. 文件結構

1. **標題**: 使用 `# 主題名稱` 格式
2. **這是什麼、為什麼需要它**: 用使用者在對話中確認過的理解來寫，不是官方定義
3. **怎麼用**: 實際程式碼和應用情境，讓回顧時能直接看到具體寫法
4. **需要注意的地方**（非必要）: 容易搞混的點、使用限制、與類似概念的差異——視討論內容決定是否需要

### 4. 儲存位置

- 目標路徑：`.claude/skills/implement-notes/references/`
- 執行前先用 Read 檢查目錄是否存在
- 若檔名已存在，詢問使用者：
  - 覆蓋現有檔案
  - 使用新檔名（如：`-v2.md` 後綴）
  - 取消操作

### 5. 完成確認

生成後回報：
- ✓ 檔案路徑：`.claude/skills/implement-notes/references/[filename].md`
- ✓ 內容摘要：簡述記錄的主題

## 品質標準

- ✓ 以使用者的理解為敘事主軸，程式碼和情境圍繞理解來組織
- ✓ 針對單一技術或邏輯，不做百科全書式整理
- ✓ 內容精簡，方便快速回顧
- ✓ 程式碼區塊標註語言
- ✓ 使用 markdown 格式化
- ✓ 中英文之間加空格（提升可讀性）
- ✓ 標題層級清晰（避免跳級）
