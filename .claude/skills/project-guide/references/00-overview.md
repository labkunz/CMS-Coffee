# 00 — 專案概述 & 技術選型

## 專案概述

**目標**：用 Contentful（Headless CMS）+ Nuxt 4 建一個咖啡廳品牌官網的最小可用版本，展示 API 串接能力，附在履歷上。

**時程**：5 天（2/17 - 2/21），22 號前完成  
**部署目標**：線上可瀏覽的完整網站

---

## 技術選型

| 層級 | 技術 | 說明 |
|------|------|------|
| CMS | Contentful（免費方案） | 雲端 Headless CMS，註冊即用，提供 REST & GraphQL API |
| 前端框架 | Nuxt 4 | 最新穩定版，SSR/SSG、app/ 目錄結構 |
| UI 套件 | Nuxt UI v4 | 125+ 免費元件，深度整合 Nuxt 4 + Tailwind CSS v4 |
| 部署 | Vercel | Nuxt 原生支援，免費方案夠用 |

---

## 為什麼選 Contentful 而不是 Strapi？

- **零部署成本**：雲端託管，註冊帳號就能用，不需要自己架伺服器或管資料庫
- **時間效率**：省下 Strapi 環境建置和部署的時間（預估省 1 天），把時間投入前端品質
- **API 品質高**：REST API 結構乾淨、文件完整，串接體驗好
- **免費方案夠用**：5 users / 25K records / 2 locales，作品集完全足夠
- **面試話術**：「評估專案需求後選擇雲端 CMS，讓團隊專注在前端交付而非基礎設施維護」

## 為什麼選 Nuxt 4 + Nuxt UI v4？

- Nuxt 4 於 2025 年 7 月正式發布，目前已到 4.3，穩定且為最新主線版本
- Nuxt UI v4 需要 Nuxt 4 才能運作，兩者綁定使用
- Nuxt UI v4 統一了原本的 Nuxt UI 和 Nuxt UI Pro，125+ 元件全部免費
- 面試展現你掌握最新版本的框架生態
